Hopefully here I can clarify what some of the perms the bot uses are, and why they are needed.


Permissions

  • MANAGE_WEBHOOKS

    • This is by far one of the bots most imprtant permissions. It's completely optional, yet also somewhat necessary. Let me explain: The bot needs this permission to view the webhooks and manage them, in some cases create new ones or delete old ones. The bot does this to ensure I can communicate in yer guild's individual channels. It will not abuse this tho.

  • VIEW_CHANNELS

    • This permission is half deprecated, used to view PUBLIC channels. I CANNOT see any private channels with this perm, besides that moderation bots ignore other bots and webhooks.

  • READ_MESSAGES

    • This is literally just equal to the READ_MESSAGE_HISTORY permission for members. Allows me to view messages up to 14 days old (API limit)

  • EMBED_LINKS

    • To have my links embed? Specially useful if ATTACH_FILES is denied, but also allows me to send ggpht.ga media via the bot.

  • ATTACH_FILES

    • Send media files. That's it...


I hope this clarified all of the perms the bot requires to function properly. The only permission that is classed as 'elevated' is MANAGE_WEBHOOKS and, freely, you can deny it that perm.

Now to explain what the bot actually does.


Bot Workings

So, how exactly does TallBot work?

    Discord Bot

  • Receive Messages

    • TallBot on Discord receives the messages. It can fully receive commands, reply to them using it's predefined commands, and shows me the message history (messages up to 14 days old) on a localhost version of BFC (modified to receive events from TallBot on Discord and Revolt)

  • Manage Webhooks

    • The bot can also manage webhooks, so that it can ensure the messages can be sent. It check them around every 23h. If they are usuable, or available where the bot can see, it checks a box, which is viewable on the localhost site. It can also move the webhooks between channels, but currently I need the guild owner or a member within a guild to send me the channel ID's.

  • Embed Media Links

    • When I want to send a link, TallBot does a check to see if it matches any image CDN I use (primarily ggpht.ga, although Firebase Storage is another one), and will send the link itself rather than via a webhook, to ensure the media is attached, or not (depending on if you allow it the permissions mentioned above!)

  • Send Media

    • Similar to the above, TallBot will also check if I'm sending media (.png, .txt, .mp4, .mp3 and .m4a only), and send said media itself, rather than webhooking that data.
